Self-service customizable CP4I environment setup
Cloud Pak for Integration (CP4I) v4 adopted the Operator based installation & management pattern. This gives us flexibility to custom build a demo environment based on our needs. You can choose to install just the required services in a self-service fashion on an appropriately sized cluster for more cost effectiveness in addition to better performance as you would have installed just services you need for your demo. This interface uses standard engineering-supported install utility/mechanism for ongoing support/maintainability.

Pre-requisites

To clear pre populated fields if any click here.

Configure Openshift credentials (either use combination of Openshift server URL, user/password (OR) Openshift server URL, token).

  1. From Openshift console, select your profile on top right and select “Copy login command”
  2. From oc cli, run “oc config view --minify|grep server” to get server name and “oc whoami -t” to get token

Obtain an IBM Software Entitlement Key from IBM Container Library using your IBM ID. Click Get an entitlement key if you are not on the entitlement page.

Instructions

Get the resources required to deploy CP4I services and connect to the openshift credentials.

Storage Vendor
Valid values include:

  • ocs: If you are using OCS storage for CP4I
  • portworx: If you are using Portworx storage for CP4I
  • roks-classic: If you want to use ibmc-file-gold-gid and ibm-block-gold-gid
